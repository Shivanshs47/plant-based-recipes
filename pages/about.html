About GreenPlate

Built for plant-based food lovers. Powered by community.

GreenPlate is a community-driven recipe platform where anyone can discover and share sustainable, plant-based recipes. Whether you're a seasoned vegan chef or just starting to explore meatless cooking, there's a place for you here.

Our Values

🌍
Sustainability
Every plant-based meal reduces your carbon footprint. Food is one of the most powerful levers for climate action.
🤝
Community
Recipes are more meaningful when shared. GreenPlate is a place for cooks of all levels to connect and inspire each other.
🥗
Nutrition
Plant-based eating doesn't mean compromising on flavour or nutrition. Our recipes prove that green food can be deeply satisfying.
💡
Education
We believe in transparency. Open the browser console to see every SQL query this app runs — the database layer is fully visible.

How It's Built

GreenPlate is a fully client-side web application built with vanilla HTML, CSS, and JavaScript — no frameworks, no build tools. The SQL database layer is simulated using JavaScript with localStorage persistence, demonstrating core database concepts in the browser.

Technology Usage File(s)
HTML5 Semantic page structure, accessibility (ARIA), forms index.html, pages/*.html
CSS3 CSS variables, Grid, Flexbox, animations, responsive design css/style.css
JavaScript (DOM) Dynamic rendering, event handling, live search, modal, toast js/app.js, js/explore.js, js/home.js, js/submit.js
SQL (Simulated) CRUD operations, WHERE/LIKE queries, ORDER BY — stored in localStorage js/db.js
Google Fonts Playfair Display (headings), DM Sans (body) Loaded via CDN

Database Schema

This is the SQL table structure used to store recipes. Open the browser DevTools console and you'll see every INSERT, SELECT, and DELETE query logged in real time.

-- GreenPlate Database Schema
-- Open DevTools Console to see live SQL queries

CREATE TABLE recipes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  category TEXT NOT NULL, -- breakfast|lunch|dinner|snacks|desserts|drinks
  difficulty TEXT NOT NULL, -- easy|medium|hard
  prep_time INTEGER, -- minutes
  cook_time INTEGER, -- minutes
  servings INTEGER,
  emoji TEXT,
  description TEXT,
  ingredients TEXT,
  steps TEXT,
  tags TEXT,
  author TEXT,
  created_at TEXT -- ISO date string
);

-- Example queries executed by this app:
SELECT * FROM recipes ORDER BY id DESC;
SELECT * FROM recipes WHERE title LIKE '%bowl%' AND category = 'lunch';
INSERT INTO recipes (title, category, ...) VALUES (?, ?, ...);
DELETE FROM recipes WHERE id = ?;
